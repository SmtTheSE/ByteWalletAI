## PRODUCTION NOTES

### Recommended Tools
- **Recording**: OBS Studio (free, open-source)
- **Editing**: DaVinci Resolve (free version) or CapCut
- **Screen Recording**: Built-in macOS QuickTime or Loom
- **Music**: Epidemic Sound or YouTube Audio Library (royalty-free)

### Visual Assets Needed
1. **Wireframes** (see accompanying Figma/mockup file)
2. **Screen recordings** of working API demo
3. **B-roll footage**: 
   - Stressed students (can use free stock from Pexels, Unsplash)
   - ASEAN cityscapes (Bangkok, Yangon, Ho Chi Minh City)
   - Mobile payment transactions
4. **Graphics**: 
   - Model architecture diagram
   - Feature importance chart
   - Regional map

### Subtitle Strategy
- Add English subtitles for accessibility
- Optional: Add Burmese/Vietnamese/Indonesian subtitles for regional reach
- Use auto-caption tools (Rev.com, Descript) then manually correct

### Presentation Tips
1. **Speak slowly and clearly** (nervous speakers tend to rush)
2. **Use hand gestures** when explaining concepts
3. **Smile during opening and closing** (creates connection)
4. **Wear professional casual** (collared shirt, solid colors work best on camera)
5. **Lighting**: Face a window for natural light, or use ring light
6. **Background**: Clean, uncluttered space or use virtual background with ByteWallet logo

### Backup Plan
If technical demo fails during recording:
- Have screenshots ready as fallback
- Use animated explainer video segment instead
- Focus on problem/solution narrative over live tech demo
